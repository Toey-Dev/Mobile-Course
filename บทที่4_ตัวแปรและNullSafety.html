1. 🖼️ ภาพเล่าเรื่อง: โกดังกล่องติดป้าย

โปรแกรมทุกโปรแกรมคือ "โกดังเก็บข้อมูล + คนงานที่หยิบของไปประมวลผล" — ตัวแปร (variable) คือกล่องติดป้ายชื่อในโกดังนั้น

🏬 โกดังของโปรแกรม Quiz ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │ 📦 question │ │ 📦 score │ │ 📦 isPassed │ │ ป้าย: String │ │ ป้าย: int │ │ ป้าย: bool │ │ ของ: "1+1=?" │ │ ของ: 10 │ │ ของ: true │ └──────────────┘ └──────────────┘ └──────────────┘ เก็บข้อความ เก็บจำนวนเต็ม เก็บ จริง/เท็จ กฎโกดัง Dart: กล่องป้าย int ห้ามยัดข้อความ! (จับได้ตั้งแต่ยังไม่รัน)

แล้ว null คืออะไร? — ฉากที่สอง

┌──────────────┐ │ 📦 nickname │ คนงานเดินมาเปิดกล่อง... │ ป้าย: String │ 😱 กล่องว่างเปล่า! (null) │ ของ: (ไม่มี) │ โปรแกรม crash ทันที └──────────────┘ ทางแก้ของ Dart: บังคับติดป้ายตั้งแต่แรกว่า "กล่องนี้อาจว่างได้" → String? ใครจะเปิดกล่องที่มีป้าย ? ต้องเช็คก่อนเสมอ — คอมไพเลอร์คอยจับตาให้
💡 เกร็ดประวัติศาสตร์

Tony Hoare ผู้คิดค้นค่า null เรียกมันว่า "ความผิดพลาดพันล้านดอลลาร์" เพราะบั๊กจากการเปิดกล่องว่าง (null error) สร้างความเสียหายมหาศาลทั่วโลกมา 50 ปี — Dart เกิดทีหลังเลยออกแบบระบบ null safety มาปิดจุดนี้ตั้งแต่ต้น นี่คือเหตุผลที่แอป Flutter crash ยากกว่าแอปยุคเก่า

2. 💻 ลงมือจริง: เริ่มโปรเจกต์ Quiz CLI

สร้างโฟลเดอร์ใหม่ชื่อ quiz_cli (อยู่นอก my_first_app) แล้วสร้างไฟล์ quiz.dart ข้างในด้วยโค้ดนี้:

// quiz.dart — จุดเริ่มต้นเกม Quiz (จะพัฒนาต่อทุกบทจนจบเฟส 2)
void main() {
  // ตัวแปรพื้นฐาน 4 ชนิดที่ใช้บ่อยสุด
  String question = 'Flutter ใช้ภาษาอะไรเขียน?'; // ข้อความ
  String answer = 'Dart';
  int score = 0;          // จำนวนเต็ม
  double version = 3.0;   // ทศนิยม
  bool isReady = true;    // จริง/เท็จ

  // String interpolation: ฝังค่าตัวแปรในข้อความด้วย $ — ใช้บ่อยมากใน Flutter
  print('=== เกม Quiz เวอร์ชัน $version ===');
  print('พร้อมเล่นไหม: $isReady');
  print('คำถาม: $question');
  print('เฉลย: $answer (ตอนนี้คะแนน: $score)');
}

รันด้วยคำสั่ง:

dart run quiz.dart
🤔 ทำไมรันด้วย dart ไม่ใช่ flutter

Dart คือภาษา ส่วน Flutter คือชุดเครื่องมือสร้าง UI ที่ใช้ภาษา Dart — โปรแกรมที่ไม่มีหน้าจอแบบนี้ใช้ dart ตรงๆ ได้เลย (ติดมากับ Flutter อยู่แล้ว) · ทางเลือกไม่ต้องติดตั้งอะไร: เว็บ dartpad.dev วางโค้ดแล้วกดรันได้ทันที เหมาะกับการทดลองสั้นๆ

var, final, const — สามระดับของ "ความเปลี่ยนแปลงได้"

var playerName = 'มายด์';     // Dart เดาชนิดให้ (String) — เปลี่ยนค่าได้
playerName = 'บีม';           // ✓ ได้

final startedAt = DateTime.now(); // กำหนดค่าได้ครั้งเดียวตอนรัน — เปลี่ยนไม่ได้
const maxQuestions = 10;          // ค่าตายตัวตั้งแต่ก่อนรัน — เปลี่ยนไม่ได้

// จำง่ายๆ:  เปลี่ยนได้ = var · ล็อกตอนรัน = final · ล็อกตั้งแต่เขียน = const

Null safety — หัวใจของบทนี้

String name = 'มายด์';   // กล่องนี้ "ต้องมีของเสมอ" — ใส่ null ไม่ได้
String? nickname;        // เครื่องหมาย ? = กล่องนี้ "อาจว่างได้"

// จะใช้กล่องที่อาจว่าง ต้องมีแผนสำรองเสมอ:
print(nickname ?? 'ยังไม่ตั้งชื่อเล่น');  // ?? = "ถ้าว่าง ให้ใช้ค่านี้แทน"

// ⚠️ เครื่องหมาย ! = "ฉันมั่นใจว่าไม่ว่าง เปิดเลย!" — ถ้าพลาด = crash
// มือใหม่ควรเลี่ยง ! แล้วใช้ ?? หรือเช็ค if แทน (บทหน้า)

3. 🧠 คอนเซปต์: ทำไมต้อง "ประกาศชนิด" ให้เข้มงวด

ภาษาบางภาษายอมให้กล่องเดียวเก็บอะไรก็ได้สลับไปมา ฟังดูสะดวก แต่เหมือน โกดังที่ไม่ติดป้าย — วันหนึ่งคนงานหยิบกล่อง "ราคาสินค้า" ไปบวกเลข แต่ข้างในดันเป็นข้อความ "สามร้อยบาท" → พังตอนลูกค้ากำลังจ่ายเงิน

Dart จับปัญหานี้ตั้งแต่ตอนเขียน (ขึ้นเส้นแดงใน VS Code ทันที) ไม่ใช่ตอนแอปอยู่ในมือผู้ใช้ — ยิ่งจับบั๊กได้เร็ว ราคาที่จ่ายยิ่งถูก:

จับได้ตอนเขียน (เส้นแดงใน editor) → ฟรี แก้ 5 วินาที จับได้ตอนรันทดสอบ → เสียเวลาไล่หา หลุดไปถึงมือผู้ใช้ → แอปได้รีวิว 1 ดาว ★

4. 🔧 เทคนิคการประยุกต์ใช้

เทคนิคที่ 1 — เริ่มด้วย final เสมอ เปลี่ยนเป็น var เมื่อจำเป็น

ตัวแปรส่วนใหญ่ในโปรแกรมจริงตั้งค่าครั้งเดียวแล้วไม่เปลี่ยน การล็อกด้วย final ทำให้ใครมาอ่านรู้ทันทีว่า "ค่านี้นิ่ง ไม่ต้องระวังใครแอบเปลี่ยน" — ลดภาระสมองตอนไล่โค้ด

เทคนิคที่ 2 — ตั้งชื่อ bool ให้อ่านเป็นประโยคคำถาม

isReady, hasAnswered, canSubmit — เห็นชื่อก็รู้ว่าตอบ ใช่/ไม่ใช่ ไม่ต้องไปดูว่าเก็บอะไร (เทียบกับ status ที่ต้องเดา)

เทคนิคที่ 3 — ใช้ ?? วางแผนสำรองให้ข้อมูลที่อาจว่าง

ทุกจุดที่ข้อมูลมาจาก "ข้างนอก" (ผู้ใช้กรอก, โหลดจากเน็ต) อาจว่างได้เสมอ นิสัยเขียน ค่าที่อาจว่าง ?? ค่าสำรอง จะติดตัวไปช่วยมากตอนทำแอปจริงเฟส 4

เทคนิคที่ 4 — interpolation ทำอะไรได้มากกว่าฝังตัวแปร

'คะแนน: ${score + 10}' — ใส่นิพจน์คำนวณใน ${...} ได้เลย ใช้จัดข้อความแสดงผลได้ยืดหยุ่นมาก

5. 🏭 ตัวอย่างการเขียนจริงในงานจริง

🏭 สถานการณ์จริง: "โปรไฟล์ผู้ใช้ที่ยังไม่ล็อกอิน"

แทบทุกแอปมีหน้าที่แสดงชื่อผู้ใช้ — แล้วถ้าผู้ใช้ยังไม่ได้ล็อกอินล่ะ? นี่คือจุดที่แอปยุคเก่า crash กันเป็นประจำ

// โค้ดจริงที่เจอในทุกแอป — จัดการ "ข้อมูลที่อาจยังไม่มี" อย่างปลอดภัย
String? loggedInUser;   // ยังไม่ล็อกอิน = null (ว่างอย่างตั้งใจ)

// จุดแสดงผล: มีแผนสำรองเสมอ ไม่มีทาง crash
print('สวัสดี ${loggedInUser ?? 'ผู้มาเยือน'} 👋');
// ยังไม่ล็อกอิน → "สวัสดี ผู้มาเยือน 👋"
// ล็อกอินแล้ว   → "สวัสดี มายด์ 👋"

ในทีมจริง ตอน code review ถ้าใครเขียน loggedInUser! (มั่นใจว่าไม่ว่าง) จะถูกถามทันทีว่า "มั่นใจได้ยังไงว่าตรงนี้ล็อกอินแล้วเสมอ?" — เครื่องหมาย ! ทุกตัวในโค้ดคือความเสี่ยงที่ต้องอธิบายได้

6. 🧹 ตัวอย่าง Clean Code

// ❌ แบบรีบๆ — ชนิดหลวม ชื่อกำกวม
var a = 'Flutter ใช้ภาษาอะไรเขียน?';
var b = 'Dart';
dynamic c = 0;        // dynamic = ปิดระบบตรวจชนิด! อะไรก็ยัดได้ = กลับไปยุคมืด
var flag = true;      // flag ของอะไร? ต้องไล่อ่านโค้ดถึงรู้
// ✅ แบบมืออาชีพ — ชนิดชัด ชื่อเล่าเรื่อง ล็อกค่าที่ไม่เปลี่ยน
final String question = 'Flutter ใช้ภาษาอะไรเขียน?';
final String correctAnswer = 'Dart';
int score = 0;                 // ตั้งใจให้เปลี่ยนได้ (คะแนนต้องเพิ่ม) เลยไม่ใส่ final
final bool isGameReady = true; // อ่านออกเสียงได้เป็นประโยค
ประเด็น❌ แบบรีบ✅ แบบ clean
ชื่อตัวแปรa, b, flag — ต้องเดาquestion, correctAnswer — เล่าเรื่องเอง
ชนิดข้อมูลdynamic ปิดระบบป้องกันระบุชัด คอมไพเลอร์ช่วยจับผิดให้
การล็อกค่าทุกอย่าง varfinal เป็นค่าเริ่มต้น — คนอ่านสบายใจ

7. ⚠️ ข้อผิดพลาดที่เจอบ่อย

① "The non-nullable local variable 'x' must be assigned before it can be used"

สาเหตุ: ประกาศกล่องแบบ "ต้องมีของ" (ไม่มี ?) แต่ยังไม่ใส่ของ แล้วรีบใช้
วิธีแก้: ใส่ค่าตั้งแต่ประกาศ (String name = '';) หรือถ้าตั้งใจให้ว่างได้ ใส่ ? (String? name;)

② "A value of type 'String' can't be assigned to a variable of type 'int'"

สาเหตุ: ยัดของผิดกล่อง เช่น int age = '25'; ('25' มีเครื่องหมายคำพูด = ข้อความ ไม่ใช่ตัวเลข)
วิธีแก้: เอาเครื่องหมายคำพูดออก (age = 25) หรือถ้าได้ข้อความมาจริงๆ แปลงด้วย int.parse('25')

③ "Null check operator used on a null value" (แอป crash)

สาเหตุ: ใช้ ! เปิดกล่องที่มั่นใจว่าไม่ว่าง... แต่มันว่าง — นี่คือ crash อันดับต้นๆ ของมือใหม่
วิธีแก้: แทบทุกครั้งที่อยากพิมพ์ ! ให้เปลี่ยนเป็น ?? ค่าสำรอง หรือเช็คก่อนใช้ · กฎง่ายๆ: "เห็น ! ในโค้ดตัวเอง = จุดเสี่ยงที่ต้องอธิบายได้ว่าทำไมปลอดภัย"

8. 🏋️ แบบฝึกหัดท้ายบท (ลงมือทำจริง)

🥉 ระดับ 1 — โกดังของฉัน

สร้างไฟล์ profile.dart ประกาศตัวแปรแนะนำตัวเอง 5 ตัว (ชื่อ String, อายุ int, ส่วนสูง double, กำลังเรียน Flutter bool, ชื่อเล่น String? ที่เป็น null) แล้ว print แนะนำตัวด้วย interpolation โดยชื่อเล่นต้องใช้ ?? แสดง "ไม่มีชื่อเล่น" แทน

🥇 ระดับ 3 — นามบัตร Terminal

เขียน card.dart พิมพ์นามบัตรของคุณในกรอบสวยงามด้วยอักขระ ═ ║ ╔ ╗ ╚ ╝ โดยข้อมูลทุกชิ้นอยู่ในตัวแปร (ห้าม hardcode ในบรรทัด print) มีอย่างน้อย 1 ฟิลด์ nullable พร้อมค่าสำรอง — เสร็จแล้ว git init โฟลเดอร์ quiz_cli แล้ว commit ทั้งหมด

9. ✅ เช็กลิสต์และสรุปบทเรียน

  • อธิบายความต่างของ String / int / double / bool พร้อมยกตัวอย่าง
  • บอกได้ว่า var / final / const ต่างกันตรงไหน และทำไมควรเริ่มจาก final
  • อธิบายได้ว่า String? ต่างจาก String ยังไง และ ?? ใช้ทำอะไร
  • รู้ว่าทำไมควรเลี่ยง ! และ dynamic
  • รัน dart run quiz.dart ผ่าน และ commit โปรเจกต์ quiz_cli แล้ว

📖 สรุปศัพท์ใหม่บทนี้

ศัพท์ความหมายสั้นๆภาพจำ

Stringข้อความกล่องเก็บตัวหนังสือ
int / doubleจำนวนเต็ม / ทศนิยมกล่องเก็บตัวเลข 2 แบบ

nullable (?)ประกาศว่า "อาจว่างได้"ป้ายเตือน "กล่องนี้อาจว่าง"

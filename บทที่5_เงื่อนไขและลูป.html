1. 🖼️ ภาพเล่าเรื่อง: ทางแยกและสายพาน

▶ โปรแกรมวิ่งมาถึง...ทางแยก (if) ▶ ...และวงเวียน (loop) คำตอบถูกไหม? เหลือคำถามอีกไหม? ┌────┴────┐ ┌──────────┐ ใช่ ✓ ไม่ ✗ │ ถามข้อถัดไป │◄──┐ │ │ │ ตรวจคำตอบ │ │ ยังเหลือ → วนต่อ score + 1 บอกเฉลย └──────┬─────┘ │ └────┬────┘ ├─────────┘ ▼ ▼ หมดแล้ว ไปข้อถัดไป สรุปคะแนน

ภาพจำ: เงื่อนไข = ทางแยกที่มีป้ายคำถาม ตอบใช่ไปทางหนึ่ง ตอบไม่ไปอีกทาง · ลูป = สายพานโรงงาน ที่วนทำงานเดิมกับของชิ้นถัดไปเรื่อยๆ จนกว่าของจะหมด

🤔 ทำไมสองเรื่องนี้ต้องมาคู่กัน

โปรแกรมเกือบทั้งหมดในโลกคือ "วนดูข้อมูลทีละชิ้น (ลูป) แล้วตัดสินใจกับแต่ละชิ้น (เงื่อนไข)" — ฟีดโซเชียลคือลูปที่วนโพสต์+เงื่อนไขว่าโพสต์ไหนควรโชว์, แอปธนาคารคือลูปวนรายการ+เงื่อนไขจัดหมวดรายรับรายจ่าย

2. 💻 ลงมือจริง: Quiz โต้ตอบได้แล้ว!

อัปเกรด quiz.dart — คราวนี้รับคำตอบจากผู้เล่นจริงผ่านคีย์บอร์ด:

import 'dart:io'; // ชุดเครื่องมือคุยกับ Terminal (stdin = รับข้อความที่ผู้ใช้พิมพ์)

void main() {
  final String question = 'Flutter ใช้ภาษาอะไรเขียน?';
  final String correctAnswer = 'dart';
  int score = 0;

  print('คำถาม: $question');
  // stdin.readLineSync() = หยุดรอให้ผู้ใช้พิมพ์แล้วกด Enter
  // ผลลัพธ์เป็น String? (อาจว่าง!) เลยต้องมี ?? — ความรู้บทที่แล้วมาใช้จริง
  final String userAnswer = stdin.readLineSync() ?? '';

  // ทางแยก: ตัดสินใจจากคำตอบ (.toLowerCase() = แปลงเป็นตัวเล็ก กันตอบ DART แล้วผิด)
  if (userAnswer.toLowerCase() == correctAnswer) {
    score += 1;   // ย่อมาจาก score = score + 1
    print('ถูกต้อง! 🎉 (+1 คะแนน)');
  } else {
    print('ยังไม่ถูก เฉลยคือ: $correctAnswer');
  }

  print('คะแนนรวม: $score');
}

รัน dart run quiz.dart แล้วลองตอบทั้งถูกและผิด — นี่คือครั้งแรกที่โปรแกรมของคุณโต้ตอบกับมนุษย์!

switch — ทางแยกหลายแพร่ง

ถ้าทางแยกมีมากกว่า 2 ทาง ใช้ switch อ่านง่ายกว่า if ซ้อนกันยาวๆ:

// ให้เกรดจากคะแนน (สมมติเต็ม 5)
final String grade = switch (score) {
  5 => 'ยอดเยี่ยม! 🏆',
  4 => 'เก่งมาก 🥈',
  3 => 'ผ่าน 🙂',
  _ => 'ลองใหม่อีกครั้งนะ 💪',  // _ = กรณีอื่นๆ ทั้งหมด (ต้องมีเสมอ)
};
print(grade);

ลูป — ถามหลายข้อโดยไม่ก็อปโค้ด

// for: รู้จำนวนรอบแน่นอน — "ทำ 3 รอบ"
for (int i = 1; i <= 3; i++) {
  print('กำลังถามข้อที่ $i จาก 3');
}

// while: ไม่รู้จำนวนรอบ — "ทำจนกว่าจะสำเร็จ"
String answer = '';
while (answer.toLowerCase() != 'dart') {
  print('ตอบผิดได้ไม่จำกัด! Flutter ใช้ภาษาอะไร?');
  answer = stdin.readLineSync() ?? '';
}
print('ในที่สุดก็ถูก! 🎉');

// break = ออกจากลูปทันที · continue = ข้ามรอบนี้ ไปรอบถัดไป

3. 🔧 เทคนิคการประยุกต์ใช้

เทคนิคที่ 1 — เลือกลูปจากคำถาม "รู้จำนวนรอบไหม?"

รู้แน่ (ถาม 10 ข้อ, วาด 5 ดาว) → for · ไม่รู้ (รอจนตอบถูก, รอจนผู้ใช้พิมพ์ "exit") → while — เลือกถูกโค้ดจะเล่าเรื่องเอง

เทคนิคที่ 2 — normalize ข้อมูลก่อนเปรียบเทียบเสมอ

ผู้ใช้พิมพ์ Dart, DART, dart (มีช่องว่าง) — ทั้งหมดควรถือว่าถูก: userAnswer.trim().toLowerCase() คือท่ามาตรฐานที่แอปจริงใช้กับ input มนุษย์ทุกจุด

เทคนิคที่ 3 — เงื่อนไขซับซ้อน ให้ตั้งชื่อก่อน

if (age >= 18 && hasIdCard && !isBanned) อ่านยาก → ตั้งชื่อก่อน: final bool canRegister = ...; แล้ว if (canRegister) — เงื่อนไขกลายเป็นประโยคภาษาคน

เทคนิคที่ 4 — กันลูปไม่มีที่สิ้นสุดด้วย "ตัวนับฉุกเฉิน"

ตอนฝึก while ใหม่ๆ เพิ่มตัวนับรอบไว้ก่อน: ถ้าวนเกิน 100 รอบให้ break — โปรแกรมค้างเพราะลูปไม่จบคือประสบการณ์ที่มือใหม่ทุกคนต้องเจอ (กด Ctrl+C เพื่อหยุดโปรแกรมที่ค้าง)

4. 🏭 ตัวอย่างการเขียนจริงในงานจริง

🏭 สถานการณ์จริง: "ระบบสมาชิก 3 ระดับ"

แอปอีคอมเมิร์ซเกือบทุกเจ้ามีสมาชิกหลายระดับ ได้สิทธิ์ต่างกัน — นี่คือ switch ในชีวิตจริง

// โค้ดสไตล์เดียวกับที่ใช้ในแอปอีคอมเมิร์ซจริง
double discountRate(String memberTier) {
  return switch (memberTier) {
    'platinum' => 0.20,  // ลด 20%
    'gold'     => 0.10,
    'silver'   => 0.05,
    _          => 0.0,   // ไม่เป็นสมาชิก = ไม่มีส่วนลด
  };
}

และลูปในชีวิตจริง: "คิดเงินตะกร้าสินค้า" — ทุกครั้งที่คุณกดสั่งซื้อ ที่หลังบ้านคือลูปวนสินค้าทุกชิ้น เช็คเงื่อนไข (ของหมดไหม? มีโปรไหม?) แล้วรวมราคา ถ้าเจอของหมดสต๊อกก็ break ออกมาแจ้งเตือนทันที — โค้ดที่คุณเพิ่งเขียนในบทนี้คือโครงเดียวกันเป๊ะ แค่เปลี่ยนจากคำถาม quiz เป็นสินค้า

5. 🧹 ตัวอย่าง Clean Code: Guard Clause — ปราการหน้าประตู

ปัญหาคลาสสิกของมือใหม่คือ "ปิรามิดแห่งความหายนะ" — if ซ้อน if ซ้อน if จนโค้ดเลื้อยไปทางขวา:

// ❌ if ซ้อนลึก — ต้องไล่อ่านทุกชั้นถึงรู้ว่าเกิดอะไร
void submitAnswer(String? answer) {
  if (answer != null) {
    if (answer.isNotEmpty) {
      if (answer.length <= 100) {
        print('ตรวจคำตอบ: $answer');   // ใจความจริงอยู่ลึกสุด!
      } else {
        print('คำตอบยาวเกินไป');
      }
    } else {
      print('กรุณาพิมพ์คำตอบ');
    }
  } else {
    print('ไม่ได้รับคำตอบ');
  }
}
// ✅ Guard clause — เช็คเงื่อนไข "ไม่ผ่าน" แล้วออกทันที เหมือน รปภ. หน้าประตู
void submitAnswer(String? answer) {
  // รปภ. 3 ด่าน: ใครไม่ผ่านเกณฑ์ เชิญกลับตั้งแต่หน้าประตู (return = จบฟังก์ชันทันที)
  if (answer == null) { print('ไม่ได้รับคำตอบ'); return; }
  if (answer.isEmpty) { print('กรุณาพิมพ์คำตอบ'); return; }
  if (answer.length > 100) { print('คำตอบยาวเกินไป'); return; }

  // ผ่านทุกด่าน = โค้ดหลักอยู่ระดับนอกสุด อ่านง่าย ไม่ต้องนับวงเล็บ
  print('ตรวจคำตอบ: $answer');
}
🧠 หลักทอง

ยิ่งใจความสำคัญอยู่ "ตื้น" เท่าไหร่ โค้ดยิ่งอ่านง่ายเท่านั้น — guard clause คือเทคนิคที่ทีมระดับโลกใช้กันเป็นมาตรฐาน และ code reviewer จะยิ้มให้คุณ

6. ⚠️ ข้อผิดพลาดที่เจอบ่อย

① โปรแกรมค้าง ไม่ตอบสนอง (Infinite Loop)

สาเหตุ: เงื่อนไขของ while ไม่มีวันเป็นเท็จ เช่น ลืมรับ answer ใหม่ในลูป
วิธีแก้: กด Ctrl+C หยุดโปรแกรม แล้วเช็คว่า "ในลูปมีอะไรที่ทำให้เงื่อนไขเปลี่ยนได้ไหม" — ทุก while ต้องมีบรรทัดที่ขยับเข้าใกล้จุดจบเสมอ

② ลูปเกิน/ขาด 1 รอบ (Off-by-one)

สาเหตุ: สับสนระหว่าง < กับ <= เช่น for (int i = 1; i < 3; i++) ได้แค่ 2 รอบ (1, 2)
วิธีแก้: ท่องว่า "เริ่ม 1 ใช้ <= จำนวนรอบ" หรือ "เริ่ม 0 ใช้ < จำนวนรอบ" · ไม่แน่ใจ? print(i) ดูเลยว่าวนกี่รอบ

③ "Conditions must have a static type of 'bool'"

สาเหตุ: ใส่สิ่งที่ไม่ใช่ จริง/เท็จ ในวงเล็บ if เช่น if (score) แทน if (score > 0)
วิธีแก้: เขียนการเปรียบเทียบให้ครบ: if (score > 0), if (name.isNotEmpty)

7. 🏋️ แบบฝึกหัดท้ายบท (ลงมือทำจริง)

🥉 ระดับ 1 — ทางแยก

เขียน grade.dart รับคะแนน (สมมติค่าในตัวแปร) แล้วใช้ if/else if/else แปลงเป็นเกรด A (≥80), B (≥70), C (≥60), F — จากนั้นลองเขียนใหม่ด้วย switch เทียบกันว่าแบบไหนอ่านง่ายกว่า

🥈 ระดับ 2 — สายพาน

ใช้ for พิมพ์สูตรคูณแม่ที่ผู้ใช้พิมพ์เข้ามา (รับด้วย stdin + int.parse) และใช้ while ทำเมนู "พิมพ์เลขแม่สูตรคูณ หรือพิมพ์ exit เพื่อจบ" วนจนกว่าผู้ใช้จะพิมพ์ exit

🥇 ระดับ 3 — Quiz 3 ข้อ + ชีวิต 3 ครั้ง

อัปเกรด quiz.dart: มีคำถาม 3 ข้อ (ใช้ตัวแปรแยกก่อน — บทที่ 7 จะสอนวิธีเก็บเป็น List ที่ดีกว่า) ผู้เล่นมี "ชีวิต" 3 ครั้ง ตอบผิดเสียชีวิต 1 ชีวิตหมด = break จบเกมทันที จบเกมสรุปคะแนน + เกรดด้วย switch — commit ด้วยข้อความ feat: add interactive questions with lives system

8. ✅ เช็กลิสต์และสรุปบทเรียน

  • เขียน if/else และ switch ได้โดยไม่เปิดตัวอย่าง
  • ตอบได้ว่าเมื่อไหร่ใช้ for เมื่อไหร่ใช้ while
  • อธิบาย guard clause ได้ว่าดีกว่า if ซ้อนยังไง
  • เคยเจอ (และแก้) infinite loop ด้วยตัวเองแล้ว
  • Quiz รับคำตอบจริง ตรวจถูกผิด และมีระบบชีวิตแล้ว

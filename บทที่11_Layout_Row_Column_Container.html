1. 🖼️ ภาพเล่าเรื่อง: มองโลกเป็นกล่อง

เปิดแอปโปรดของคุณตอนนี้ แล้วลองมองใหม่: ทุกหน้าจอคือกล่องซ้อนกล่อง — ไม่มีข้อยกเว้น

สิ่งที่ตาเห็น สิ่งที่ dev เห็น (X-ray vision) ┌─────────────────────┐ ┌─ Column (กล่องเรียงแนวตั้ง) ──┐ │ [รูป] ชื่อร้าน ⭐4.9 │ │ ┌─ Row (เรียงแนวนอน) ──────┐ │ │ │ │ │ [รูป] [Column ชื่อ+ดาว] │ │ │ ═══ เมนูแนะนำ ═══ │ │ └──────────────────────────┘ │ │ ☕ ลาเต้ 55.- │ │ ┌─ Row ────────────────────┐ │ │ 🍰 เค้ก 89.- │ │ │ [ไอคอน] [ชื่อ] [ราคา] │ │ │ │ │ └──────────────────────────┘ │ └─────────────────────┘ └──────────────────────────────┘ กฎเดียวที่ต้องจำ: เรียงลงล่าง = Column · เรียงไปขวา = Row ทุกดีไซน์ในโลกแตกเป็นสองอย่างนี้ซ้อนกันได้เสมอ

แล้วแกนหลัก-แกนขวาง (main/cross axis) คืออะไร? — ฉากที่สอง

Column: แกนหลักวิ่งลง ⬇ Row: แกนหลักวิ่งไปขวา ➡ ┌───────────┐ ┌─────────────────────────┐ │ [ A ] │ mainAxis = │ [A] [B] [C] │ │ [ B ] │ แนวตั้ง └─────────────────────────┘ │ [ C ] │ mainAxis = แนวนอน └───────────┘ crossAxis = crossAxis = แนวตั้ง แนวนอน mainAxisAlignment = จัดเรียงตามแกนหลัก (ชิดบน/กลาง/ล่าง หรือ ซ้าย/กลาง/ขวา) crossAxisAlignment = จัดเรียงตามแกนขวาง (แกนที่เหลืออีกแกน)
🤔 ทำไมต้องเข้าใจสองแกนนี้ให้แน่น

คำถาม layout ของมือใหม่ 80% ("ทำไมมันไม่อยู่กลาง", "ทำไมชิดซ้ายหมด") ตอบได้ด้วยการถามกลับว่า "คุณกำลังจัดแกนไหนอยู่?" — mainAxisAlignment กับ crossAxisAlignment จัดคนละแกนกัน สลับกันเมื่อไหร่ งงเมื่อนั้น

2. 💻 ลงมือจริง: สามผู้ช่วยจัดกล่อง + Header Card ของ To-Do

Container — กล่องอเนกประสงค์ (ใส่สี ขอบ มุมโค้ง ระยะห่าง)

Container(
  padding: const EdgeInsets.all(16),        // ระยะห่าง "ข้างใน" กล่อง
  margin: const EdgeInsets.only(bottom: 12), // ระยะห่าง "ข้างนอก" กล่อง
  decoration: BoxDecoration(
    color: Colors.teal.shade50,
    borderRadius: BorderRadius.circular(12), // มุมโค้ง
  ),
  child: const Text('ฉันอยู่ในกล่องสวยๆ'),
)

ภาพจำ padding vs margin: กล่องพัสดุ — padding = กันกระแทกในกล่อง (ของกับผนังกล่อง) · margin = ระยะระหว่างกล่องกับกล่องข้างๆ

SizedBox — ช่องว่างที่ระบุขนาดแน่นอน

Column(children: [
  Text('บรรทัดแรก'),
  SizedBox(height: 8),    // เว้น 8 พิกเซล — วิธีเว้นช่องที่ใช้บ่อยที่สุด
  Text('บรรทัดสอง'),
])

Expanded — "ขอพื้นที่ที่เหลือทั้งหมด"

Row(children: [
  Icon(Icons.task_alt),
  SizedBox(width: 8),
  Expanded(                       // ← ดันข้อความให้กินพื้นที่ที่เหลือ
    child: Text('ชื่องานที่อาจยาวมากกกกกก...'),  // ยาวเกิน = ตัดขึ้นบรรทัดใหม่ ไม่ล้นจอ
  ),
  Text('14:00'),                  // เวลาชิดขวาพอดี
])

ประกอบร่าง: Header Card ให้ To-Do (เพิ่มใน build ของ _HomeScreenState)

// วางไว้บนสุดของ body — สรุปสถานการณ์งานวันนี้ในกล่องเดียว
Container(
  margin: const EdgeInsets.all(16),
  padding: const EdgeInsets.all(20),
  decoration: BoxDecoration(
    color: Colors.teal,
    borderRadius: BorderRadius.circular(16),
  ),
  child: Row(
    children: [
      const Icon(Icons.checklist, color: Colors.white, size: 40),
      const SizedBox(width: 16),
      Expanded(
        child: Column(
          crossAxisAlignment: CrossAxisAlignment.start,  // ชิดซ้ายตามแกนขวาง
          children: [
            const Text('งานวันนี้',
                style: TextStyle(color: Colors.white70, fontSize: 14)),
            Text('${_tasks.length} รายการ',
                style: const TextStyle(
                    color: Colors.white, fontSize: 24, fontWeight: FontWeight.bold)),
          ],
        ),
      ),
    ],
  ),
)

อ่านโครง "จากนอกเข้าใน" (เทคนิคบทที่ 1): Container (กล่องสีเขียว) → Row (ไอคอน | ข้อความ) → Column (ข้อความ 2 บรรทัด) — hot reload แล้วเห็นการ์ดสรุปงานสวยๆ ทันที

3. 🔧 เทคนิคการประยุกต์ใช้

เทคนิคที่ 1 — วาดกรอบบนกระดาษก่อนเขียนโค้ด

เจอดีไซน์ที่อยากทำ ให้หยิบปากกาตีกรอบ: อะไรเรียงลงล่าง = Column อะไรเรียงข้าง = Row ตีเสร็จค่อยพิมพ์ — มือโปรทำแบบนี้จริงๆ เร็วกว่านั่งเดาในหัวมาก

เทคนิคที่ 2 — ข้อความใน Row ให้ครอบ Expanded ไว้ก่อนเสมอ

ข้อความจากผู้ใช้/ข้อมูลจริงยาวแค่ไหนไม่มีใครรู้ — Expanded ทำให้มันตัดบรรทัดแทนที่จะล้นจอ (นี่คือวัคซีนป้องกัน error อันดับ 1 ของบทนี้ล่วงหน้า)

เทคนิคที่ 3 — ใช้ Widget Inspector ตอบคำถาม "ทำไมมันอยู่ตรงนั้น"

เครื่องมือจากบทที่ 3 กลับมาสำคัญมาก: คลิกองค์ประกอบที่วางตัวแปลกๆ → Inspector โชว์ขอบเขตกล่องจริง → เห็นทันทีว่ากล่องไหนกินพื้นที่เกิน/ขาด

เทคนิคที่ 4 — เว้นช่องด้วย SizedBox ให้เป็นค่าที่มีระบบ

เลือกชุดค่าตายตัว (เช่น 4, 8, 16, 24) แล้วใช้แค่ชุดนี้ทั้งแอป — UI จะดู "เป็นระเบียบอย่างอธิบายไม่ถูก" นี่คือความลับพื้นฐานของดีไซน์ระบบ (design system) ที่ทีมจริงใช้

4. 🏭 ตัวอย่างการเขียนจริงในงานจริง

🏭 สถานการณ์จริง: "แปลงแบบจาก Figma เป็นโค้ด"

งานประจำของ Flutter dev คือรับแบบจากดีไซเนอร์ (ไฟล์ Figma) มาแปลงเป็นจอจริง — ขั้นตอนที่ทีมจริงใช้:

① เปิดแบบ → ตีกรอบใหญ่สุดก่อน: ทั้งหน้าเรียงลงล่าง = Column ② ไล่ทีละแถบ: แถบไหนมีของเรียงข้างกัน = Row ③ ระบุ "ใครยืด ใครคงที่": รูปโปรไฟล์คงที่ 48px, ชื่อยืด (Expanded), เวลาคงที่ ④ วัดระยะห่างจากแบบ → แปลงเป็น SizedBox/Padding ตามชุดค่า 4/8/16/24 ⑤ เขียนโค้ดจากนอกเข้าใน + hot reload เทียบกับแบบทุก 2-3 บรรทัด

การ์ดสรุปที่คุณเพิ่งเขียนคือผลลัพธ์ของกระบวนการนี้เป๊ะ — ฝึกวงจรนี้จนชิน แล้ว "แปลงแบบเป็นจอ" จะกลายเป็นทักษะขายได้ของคุณ

5. 🧹 ตัวอย่าง Clean Code: แตกกล่องลึกเป็น Widget ย่อย

Layout ซ้อนลึกคือธรรมชาติของ Flutter แต่ "ซ้อนลึกในไฟล์เดียว" คือทางสู่โค้ดอ่านไม่ออก:

// ❌ build เดียว 80 บรรทัด — วงเล็บปิดไล่ไม่ถูก
Widget build(BuildContext context) {
  return Scaffold(
    body: Column(children: [
      Container( /* header 25 บรรทัด */ ),
      Container( /* summary 20 บรรทัด */ ),
      Column( /* รายการงาน 30 บรรทัด */ ),
    ]),   // ← บรรทัดนี้ปิดอะไรบ้าง? ไม่มีใครอยากนับ
  );
}
// ✅ แตกเป็น Widget ย่อยชื่อสื่อความหมาย — build กลายเป็นสารบัญ (แบบ main() บทที่ 6)
Widget build(BuildContext context) {
  return Scaffold(
    body: Column(
      children: [
        _TaskSummaryCard(taskCount: _tasks.length),  // อ่านชื่อก็รู้ว่าคืออะไร
        _TaskListSection(tasks: _tasks),
      ],
    ),
  );
}

/// การ์ดสรุปงาน — Stateless เพราะรับข้อมูลมาแสดงเฉยๆ (ตารางบทที่ 10)
class _TaskSummaryCard extends StatelessWidget {
  const _TaskSummaryCard({required this.taskCount});
  final int taskCount;

  @override
  Widget build(BuildContext context) {
    return Container( /* โค้ด header card จากหัวข้อ 2 */ );
  }
}
🧠 กฎหัวแม่มือ

Widget ซ้อนเกิน ~4 ชั้น หรือ build ยาวเกิน ~40 บรรทัด = ถึงเวลาแตก Widget ย่อย — ใน VS Code คลิกขวาที่ Widget → Refactor → Extract Widget ทำให้อัตโนมัติเลย

6. ⚠️ ข้อผิดพลาดที่เจอบ่อย

① แถบเหลือง-ดำ "BOTTOM OVERFLOWED BY 42 PIXELS"

สาเหตุ: ของในกล่องใหญ่กว่ากล่อง — Column/Row มีลูกเกินพื้นที่จอ
วิธีแก้: ถามก่อนว่า "อะไรควรหด/เลื่อนได้" — ข้อความยาว → ครอบ Expanded · เนื้อหายาวทั้งจอ → เปลี่ยนเป็นเลื่อนได้ (บทหน้าเจอ ListView พอดี) · แถบเหลือง-ดำโผล่เฉพาะตอน dev — แต่อย่าปล่อยผ่าน มันคือบั๊กจริง

② "Incorrect use of ParentDataWidget"

สาเหตุ: ใช้ Expanded นอก Row/Column — Expanded คือคำสั่ง "ขอพื้นที่จากแม่" มันเลยต้องเป็นลูกตรงของ Row/Column เท่านั้น (วางใน Container/Center = พัง)
วิธีแก้: เช็คว่าแม่ตรงของ Expanded คือ Row หรือ Column ไหม ถ้าไม่ใช่ ย้ายหรือถอดออก

③ จัดกึ่งกลางไม่ได้สักที / Container ไม่ยอมกว้างเต็มจอ

สาเหตุ: กล่องหลายชนิด "หดตามลูก" โดยธรรมชาติ — Container ที่ไม่กำหนดขนาดจะใหญ่เท่า child เท่านั้น
วิธีแก้: อยากกว้างเต็มจอ → width: double.infinity · อยากอยู่กลางจอ → ครอบด้วย Center · ไม่แน่ใจว่ากล่องไหนขนาดเท่าไหร่ → Widget Inspector ดูขอบเขตจริงทันที

7. 🏋️ แบบฝึกหัดท้ายบท (ลงมือทำจริง)

🥉 ระดับ 1 — เล่นแกน

สร้างหน้าทดลอง: Column ที่มีกล่องสี 3 ใบ (Container 60×60 คนละสี) แล้วลองสลับค่า mainAxisAlignment (start/center/spaceBetween/spaceEvenly) และ crossAxisAlignment ทีละค่า สังเกตผลด้วย hot reload — จากนั้นเปลี่ยน Column เป็น Row แล้วดูว่าแกนสลับกันยังไง

🥈 ระดับ 2 — Header Card

ใส่ header card จากหัวข้อ 2 เข้าแอป To-Do จริง แล้วอัปเกรด: เพิ่มข้อความวันที่ (มุมขวาบนของการ์ด) และทำให้ตัวเลขจำนวนงานอัปเดตเมื่อกดปุ่มเพิ่มงาน — commit ด้วย feat: add summary header card

🥇 ระดับ 3 — นามบัตรดิจิทัลกลับมาแล้ว

จำแบบฝึกหัดบทที่ 1 ได้ไหมครับ — คราวนี้ทำหน้า "โปรไฟล์ผู้ใช้" เต็มรูปแบบ: รูปโปรไฟล์ (CircleAvatar), ชื่อ, คำอธิบาย, และแถวสถิติ 3 ช่อง (งานทั้งหมด/เสร็จ/ค้าง — ใช้ Row + Expanded แบ่ง 3 ส่วนเท่ากัน) จัดระยะด้วยชุดค่า 8/16/24 เท่านั้น แตกเป็น Widget ย่อยอย่างน้อย 2 ตัว — commit ด้วย feat: add profile page layout

8. ✅ เช็กลิสต์และสรุปบทเรียน

  • มองหน้าจอแอปจริงแล้ว "ตีกรอบ" เป็น Row/Column ในหัวได้
  • อธิบายความต่าง mainAxisAlignment กับ crossAxisAlignment ได้
  • อธิบายความต่าง padding กับ margin ด้วยภาพกล่องพัสดุได้
  • รู้ว่าเมื่อไหร่ต้องใช้ Expanded และทำไมมันอยู่นอก Row/Column ไม่ได้
  • เคยเจอ (และแก้) แถบเหลือง-ดำ overflow ด้วยตัวเองแล้ว
  • To-Do มี header card และแตก Widget ย่อยแล้ว

📖 สรุปศัพท์ใหม่บทนี้

ศัพท์ความหมายสั้นๆภาพจำ
Rowเรียงลูกในแนวนอนรถไฟ
Columnเรียงลูกในแนวตั้งลิฟต์
Containerกล่องอเนกประสงค์ (สี ขอบ มุมโค้ง ระยะ)กล่องพัสดุแต่งได้
Padding / EdgeInsetsระยะห่างข้างในกล่องกันกระแทกในกล่อง
SizedBoxกล่องเปล่าขนาดตายตัว ไว้เว้นช่องบล็อกไม้คั่นระยะ
Expandedขอพื้นที่ที่เหลือทั้งหมดจากแม่ฟองน้ำขยายเต็มช่อง
main / cross axisแกนหลัก / แกนขวาง ของ Row/Columnทางรถไฟ / ไม้หมอน
🎓 สรุปบทนี้ใน 3 บรรทัด

ทุกหน้าจอคือกล่องซ้อนกล่อง — ลงล่าง Column ไปขวา Row · จัดแกนให้ถูกตัว (main vs cross) · ข้อความใน Row ครอบ Expanded กันล้น · ซ้อนเกิน 4 ชั้นให้แตก Widget ย่อย
